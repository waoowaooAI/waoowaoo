# iVibeMovie PRD v1（冻结版）

## 1. 产品目标

构建一个面向个人创作者的 AI 短剧创作平台，完成从“文本输入”到“时间轴剪辑导出”的端到端闭环，强调：

1. 可控：规则可执行、失败可追踪、状态可恢复。
2. 可编辑：全流程支持局部修改与重生成。
3. 可量化：有明确质量 KPI 与发布门槛。

## 2. 核心用户与场景

### 2.1 首发用户

- 个人创作者（Primary Persona）。

### 2.2 典型场景

1. 输入题材与剧情描述，上传小说文本（可选）。
2. 设置片段时长、每集时长、总时长。
3. 通过 AI 拆分与生成资产、分镜、视频。
4. 在时间轴中剪辑并导出。

## 3. 范围（In Scope）

1. 新建项目与时长公式校验。
2. AI 剧集拆分与片段拆分。
3. 统一上下文生成。
4. 角色/场景/道具提取与资产生成。
5. 分镜脚本生成（秒级区间 + 台词 + 音效 + 语气）。
6. 分镜图生成与替换（默认 9 宫格）。
7. 片段视频生成/重生成/上传/下载。
8. 时间轴拼接、裁剪、拖动、预览、导出。
9. PostgreSQL + PG 队列 + 内置运维监控。

## 4. 范围（Out of Scope）

1. 不保留旧接口与旧行为兼容层。
2. 不迁移历史 MySQL 数据。
3. 不保留 Redis/BullMQ/bull-board（3010）。
4. 不在 MVP 阶段引入复杂企业级 RBAC。

## 5. 强约束

1. 数据库统一 PostgreSQL。
2. 表名与字段名统一 snake_case。
3. 所有表/字段必须有中文注释。
4. 错误处理遵循零隐式回退：禁止静默兜底。
5. 前端统一 shadcn ui + Tailwind CSS。

## 6. 核心流程（11 Stage）

1. Stage 1：项目输入与时长校验。
2. Stage 2：AI 拆分剧集。
3. Stage 3：AI 拆分片段。
4. Stage 4：AI 生成统一上下文。
5. Stage 5：AI 提取角色/场景/道具。
6. Stage 6：资产生成工作台（上传/重生成/重做）。
7. Stage 7：角色三视图 + 场景多视图。
8. Stage 8：分镜脚本（秒级区间）。
9. Stage 9：分镜图默认 9 宫格。
10. Stage 10：片段视频资产化。
11. Stage 11：时间轴剪辑与导出。

## 7. 质量 KPI（发布门槛）

1. 结构化输出合规率 >= 95%。
2. 核心流程成功率 >= 90%。
3. 任务恢复成功率 >= 99%。

## 8. 发布准入

发布前必须满足：

1. 核心流程 Stage 1-11 全链路打通。
2. 合约测试、回归测试、稳定性测试通过。
3. 无 Redis/BullMQ/3010 残留运行依赖。
4. 品牌残留（ivibemovie）在核心用户路径中为 0。

## 9. 回滚策略

1. 接口层：仅保留 v2，异常通过 feature flag 禁用新入口并阻断提交。
2. 任务层：暂停 worker 消费，保留已入库任务与事件日志。
3. 数据层：按发布批次备份 schema 与关键业务表。
