    "characters": [

Planning rules:
1. Keep strict chronological order.
2. Avoid missing key beats from clip_content.
3. Keep panel transitions smooth and logically continuous.
4. Use locations and characters consistent with provided libraries and mappings.
5. Copy characterId and appearanceId exactly from the character appearance list. Do not invent IDs.
6. Prefer concrete, visible actions over abstract wording.
7. If location context contains available slots, treat them as preferred placement anchors rather than mandatory hard boundaries.
8. Prefer assigning a `slot` by copying one full placement phrase from `available_slots` verbatim when a character is stably positioned in the scene.
9. You may omit `slot` when the shot is mainly about movement, entry/exit, path traversal, transition space, temporary space, empty space, imagination, dream, memory, or abstract/non-literal space.
10. When no `slot` is used, decide placement freely from source text, action flow, and cinematic staging instead of forcing one of the provided slots.
11. Do not shorten, rewrite, summarize, or replace a provided slot phrase with a short token such as `slot_1`, `left`, or `throne_front`.
12. Do not overuse direct-to-camera shots. Unless the source explicitly requires looking into the camera, a selfie, interview, surveillance frame, or subjective gaze at the audience, avoid phrases such as "looking into camera", "staring at camera", or "facing camera". Prefer shot/reverse shot, over-the-shoulder, profile, three-quarter profile, gaze toward a scene partner, or gaze toward an off-screen target.
13. Return strict JSON only.
14. ⚠️ JSON SAFETY: All quotation marks in text (""''「」 etc.) MUST be converted to corner brackets「」in JSON string values. NEVER use raw ASCII double quotes " inside string values.
